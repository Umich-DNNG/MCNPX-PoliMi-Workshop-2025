The Challenge of Low-Energy Extrapolation

Standard Practice

Organic scintillators like EJ-309 are standard for fast neutron detection in safeguards and physics. Accurate simulation requires a precise understanding of the non-linear light output function, L(E).

The Problem

Most empirical models (polynomials, simple exponentials) fit high-energy data well but diverge significantly when extrapolating to low energies (< 100 keV). This leads to errors in simulated efficiency and threshold calculations.

Simulation Geometry

MCNPX-PoliMi Configuration

  • Source: Cf-252 spontaneous fission source.
  • Distance: Placed approximately 116 cm from the detector face.
  • Detector: Cylindrical EJ-309 liquid scintillator (12.7 cm diameter x 12.7 cm thickness).
  • Code: MCNPX-PoliMi tracks individual collision events, enabling the application of non-linear light output to multiple scatter events.

Physics Setup: SRIM Input

Defining Material Properties

To accurately model quenching, we must first calculate the stopping power (dE/dx) of the scintillator material.

  • Software: SRIM (Stopping and Range of Ions in Matter).
  • Target: EJ-309 (H:C ratio ~ 1.25).
  • Density: 0.959 g/cm³.
  • Ion: Hydrogen (Recoil Protons).

Physics Data: Stopping Power

Integration into Code

The output table provides Electronic and Nuclear stopping powers in MeV/(mg/cm²).

L(E) = ∫ S⋅dE 1+kB⋅(dE/dx)

These values are exported to EJ309.txt and read by the post-processor to calculate light yield dynamically for every collision event.

Experimental Validation (PSD)

Isolating the Neutron Signal

We employ Charge-Integration Pulse Shape Discrimination (PSD) to separate neutrons from gamma rays.

  • Upper Band: Neutrons (characterized by a larger tail integral).
  • Lower Band: Photons (smaller tail integral).
  • Analysis: Only pulses falling above the red discrimination line are compared against the simulation.
Fig 3. Measurement data from M.A. Norsworthy et al., NIMA 842 (2017) 20-27

Simulation vs. Experiment

Fractional Difference

We compared the fractional difference (S-M)/M between Simulated and Measured Pulse Height Distributions (PHDs).

Exponential models show significant under-prediction (up to 30% error) at low energies. The Birks model (red line) provides the best performance, with bin-by-bin agreement better than 5% over the range 0.08–2.18 MeVee.

Practical Implementation

Step 1: Simulation

Run MCNPX-PoliMi with the specific input deck (EJ309.i) to generate the collision dump file.

Step 2: Config

Define material properties in input_config.toml. Set kB = 33.84 and S = 2.277.

Step 3: Post-Process

Run pyMPPost. The code reads the collision file, applies the Birks function, and generates the final PHD.
